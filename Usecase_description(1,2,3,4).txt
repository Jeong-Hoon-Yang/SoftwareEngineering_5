Use case Description
1. 회원 가입
 - 사용자는 사이트에서 기본 정보(이름, 주민번호, 주소, 이메일 등)을 입력
 - 시스템은 해당 사용자가 안전한 사용자인지 인증 과정을 거쳐 정상적인 사용자인지 확인     후 시스템 사용 권한 부여
 - 회원 가입 완료

2. 회원 탈퇴
 - 사용자는 개인 정보 확인 화면의 탈퇴 버튼 누름
 - 시스템은 해당 사용자의 판매 내역 조회 (판매 등록한 상품이 남아 있으면 ①로 없으면 ②     로) 
   ①인 경우 – 사용자에게 판매 등록한 상품을 취소하라는 메세지를 보냄
   ②인 경우 – 사용자에게 정말 탈퇴할 것 인지를 물어보는 메세지를 출력
 - 사용자는 ①인 경우 판매 중인 상품을 등록 취소 ②인 경우는 실제로 탈퇴 할 것임을 알     림
 - 시스템은 ①인 경우 사용자의 판매 중인 상품을 조회 후 없는 것이 확인되면 정말 탈퇴      할 것인지를 다시 물어봄
   시트템은 ②의 경우 정말로 탈퇴할 것을 확인 한 후 사용자의 기본 정보를 삭제 후 사용     권한 소멸시킴
 - 사용자는 ①의 경우 시스템이 보내는 마지막 확인 메세지에 대해 탈퇴할 것이라고 알림
 - 시스템은 ①의 경우 사용자의 의사 확인 후 사용자의 기본 정보를 삭제 후 사용 권한을      소멸시킴
 
3. 로그인
 - 사용자는 회원 가입 시 등록한 ID와 Password를 화면에 입력
 - 시스템은 입력 받은 ID와 Password가 맞는지 확인 틀릴 시 재입력 메세지 출력 (이후 확     인되면 이후의 기능을 사용할 수 있도록 넘어감

4. 로그아웃
 - 사용자는 설정 화면에서 로그아웃을 누름
 - 시스템은 로그아웃 후 시스템을 종료함


5. 판매 의류 등록
 - 사용자는 판매 등록 페이지로 이동하여 판매 등록 하고자 하는 상품의 정보(상품명, 제작     회사명, 가격, 수량, 추가상품 가격, 판매 종료일 등)등을 입력
 - 시스템은 입력 받은 정보를 DB에 저장 후 완료 메세지 출력

6. 판매 의류 조회
 - 사용자는 개인 페이지에서 판매 중인 의류 상품 리스트로 들어감
 - 시스템은 해당 사용자가 판매 중인 상품을 DB에서 찾은 후 정렬 기준에 맞춰 출력

7. 판매 의류 수정
 - 사용자는 판매 의류 조회 페이지에서 받은 목록 중 수정하고자 하는 상품 페이지로 이동
 - 시스템은 해당 상품 정보들을 출력
 - 사용자는 해당 상품 정보들 중 수정하고자 하는 내용을 입력하고 확인 버튼을 누름
 - 시스템은 해당 입력 받은 정보를 DB에 다시 저장 후 완료 메세지 출력

8. 판매 의류 삭제
 - 사용자는 판매 의류 조회 페이지에서 받은 목록 삭제하고자 하는 상품 클릭
 - 시스템은 해당 상품을 삭제 할지 물어보는 메세지 출력 정말 삭제를 원하면 DB에서 해당     상품 정보를 지우고 완료 메세지 출력
   사용자에게 해당 상품이 지워진 판매 의류 조회 페이지 출력
